# О программе

## Реализованные модели

Реализованы классы `Currncy` и `CurrencyPair`.

`Currency` является `enum` с захардкоженными возможными валютами. Так же в этом классе определён метод для получения валюты, используя её символ.

`CurrencyPair` представляет из себя `data` класс с двумя фабричными методами, которые возвращают объект валютной пары. Я решил реализовать его именно `data` классом, посколько в классе обменника `Exchanger` я использую `Map` для хранения валютных пар и их цены. Чтобы у меня была возможность искать по ключу необходимую валютную пару, то следует переопределить методы `equals` и `hashCode`, что за меня автоматически делает `data` класс.

## Исключения

Реализованы 3 дополнительных класса исключений: `InsufficientFundsException`, `NoSuchCurrencyException`, `UnsupportedCurrencyPairException`.

Реализованы они для более конкретного определения типа возникающих исключений.

## Основные компоненты

`User` - класс пользователя, реализует интерфейс `UserProvider`, сделано так для возможности ещё создавать классы пользователей, но при этом не меняя логику уже написанной программы. При создании объекта пользователя, в конструкторе ему задаётся начальное значение валюты. Все приобретённые валюты пользователь хранит у себя в `MutableMap`. У пользователя определён метод `getCurrencies`, который возвращает неизменяемую `Map` для просмотра всех имеющихся валют у пользователя.

`Exchanger` - класс обменника. Реализует интерфейс `ExchangerProvider`, сделано так для возможности ещё создавать классы обменников. Цена валютной пары хранится в `BigDecimal` с 3 знаками после запятой. Сделать я решил именно так, поскольку `BigDecimal` позволяет точно работать с числами с плавающей точкой, и такая реализация мне показалась более удобной в отличие от реализации с целыми числами. В обменнике я хардкожу возможные валютные пары, и в блоке `init` выполняется смен курса для всех валютных пар. Функция обмена принимает любую реализацию интерфейса `UserProvider`, базовую валюту, количество базовой валюты и котируемую валюту. После успешного обмена корс валют снова меняется.

`Main` - в классе реализована обычное меню с выбором действий, присутствует проверка не некорректный ввод, а так же EOF.
